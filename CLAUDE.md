# Crystal Command

A tiny browser RTS in the spirit of Command & Conquer / StarCraft 2, built for and with Bronson (a big fan of those games). Pure canvas + vanilla JS, zero dependencies, runs from `file://` by double-clicking `index.html`.

- **Repo:** https://github.com/bronsongannon/crystal-command (public)
- **Live / shareable:** https://bronsongannon.github.io/crystal-command/ — GitHub Pages serves the `main` branch root, so **every push to `main` auto-deploys** (~1 min).

## Files

- `index.html` — page shell, all CSS, UI DOM (top bar, minimap, command card, help, toasts, game-over overlay)
- `game.js` — the entire game (~1,300 lines): data tables, fog of war, economy, combat, enemy AI, input, rendering, sprite FX, WebAudio sfx
- `assets/fx/` — Kenney CC0 particle sprites (explosion/smoke/puff variants + two muzzle-flash shapes), downscaled from the source packs. Loaded by the `loadSprites` block in `game.js`; if any image fails, `spritesReady` stays false and effects fall back to the original procedural drawing (game still works from a bare `game.js` + `index.html`).
- `assets/sprites/` — Kenney CC0 body sprites for all units & buildings (tanks pack + Top-Down Shooter infantry + Tower Defense building plates/turret gun). Loaded by `loadBodies`; `bodiesReady` gates sprite vs procedural rendering the same way. Team coloring is done at load via `teamSprite()` (multiply-tint cache) — art is neutral sand/khaki, tinted teal/red per team.
- `README.md` — player-facing docs (controls, units); keep it updated when gameplay changes

## Game snapshot (as of 2026-07-09)

- **Loop:** pick map + difficulty at the start menu → harvest crystals → train army → survive escalating AI assault waves → crack dino nests to expand → destroy enemy HQ.
- **Units:** Harvester (60) · Engineer (90, repairs buildings, trained from HQ, auto-repairs nearby damage when idle) · Marine (80, `H` toggles hunker: ½ damage taken, holds position, still shoots — `order.type==='hunker'`) · Sniper (130, prone silhouette, longest sight) · Raider (150, fast wedge buggy) · Tank (220) · **Artillery (270, 2026-07-09)**: siege RPS counter — `kind:'arc'` shells fly to where the target *was* (no homing, `target:null`) + splash 40 + 1.5× vs buildings; `minRange` 90 dead zone; sight 230 < range 300 so it wants spotters; card slot D (prodKeys now Q/W/E/R/D); can hunker like marines (`canHunker`, 2026-07-09) — ½ damage, holds position, dead zone still applies while dug in. Turret (140) must anchor within `PLACE_NEAR_BASE` of a friendly NON-turret building (anti-turret-creep, playtest feedback). Units wall-slide around buildings (`moveToward` lookahead steer, playtest feedback: was "grinds forever", now ~0.25s overhead).
- **Dinos (2026-07-09, team 3, neutral):** `nest` building (850hp, no team-2 style AI) guards each rich patch; keeps `NEST_BROOD` (3) Spitters alive, respawning one per `NEST_RESPAWN` (7s, clock only runs while short). Spitters use `order.type==='guard'` — aggro within range+90, chase to `NEST_LEASH` (360) of home, walk back; they never leave guard, so artillery beyond aggro range kills nests risk-free (the intended synergy). Fog/targeting code uses `team !== 1` (not `=== 2`) everywhere hostiles are hidden. Dinos draw procedurally (`drawDino`/`drawNest`) even when sprites are on — swap in art later via `drawUnitSprite`.
- **World 96×72 tiles (2026-07-10, was 64×48):** 3072×2304 px — base-to-base ~3250px. MAPS coords are W/H-relative so they scaled automatically; ground texture density scales by area.
- **Terrain (2026-07-10):** `rocks[]` circles from authored `ridges`/`boulders` per MAPS entry, expanded in setup, pre-rendered into the ground canvas (`paintRock`). Impassable to ground units (separation + placement checks); flyers ignore. **Pathfinding**: tile grid (`blocked`, rocks inflated +22px so tanks fit) + LOS fast path + cached A* per order (`o._path`, smoothed). Watchdog: no goal-progress for 2.5s → re-path + `ghostT` 60 (unit briefly ignores rock AND building separation to escape concave pockets — the two-circle pinch and the rock+nest pinch are otherwise inescapable). GOTCHA: map design must keep ridge ends ≥200px from patches/nests — buildings aren't in the path grid, so a ridge ending at a nest creates a blind pocket. Verified 24/24 crossings on all maps (early "failures" were marines eaten by nest guards mid-route).
- **Rocket Trooper (2026-07-10):** Barracks (140). `vehBonus: 1.6` applied at fire() vs `isVehicle` targets; hits air. Barracks now has 6 card slots — prodKeys grew KeyZ (Q/W/E/R/D/Z).
- **APC (2026-07-10):** Factory (200), `cargo: 4`. `order 'board'` (right-click friendly APC with infantry; engineers repair it instead when damaged), `unloadAPC` (key U / card button). Boarded units leave `units[]` but still count supply (`supplyUsed` sums cargo); cargo dies with the APC; pruneSelection/group-recall exclude boarded units (units.includes check).
- **Harrier (2026-07-10):** Airpad (320). `u.armed` + orders `'strike'` (fly to target, one bomb: 120 dmg ×1.4 bld, splash 55) and `'rearm'` (`HARRIER_REARM` 7s at any own airpad). idle/attackmove convert to strike when armed; `commandAttack` routes harriers to strike. AI trains after 9 min. `HARRIER_CAP` 5 per side (alive + queued), enforced in trainUnit; card shows fleet count.
- **Nukes (2026-07-10):** `silo` BLD (500, key N) + `NUKE` spec (tac 10k: r170 dmg1300 hqSafe; hq 25k 'Bunker Buster': r200 dmg3200 kills HQs). `buyNuke`/`launchNuke`/`detonate` (friendly fire on, 45% falloff at rim, tac skips HQs entirely); `NUKE_HQ_EXCLUSION` 180 blocks AIMING tac at an HQ. 30s countdown (`nukes[]`), pulsing world marker + countdown drawn ABOVE fog + blinking minimap dot + 5s warning. Player UX: silo card buys (Q/W), L → `nukeTargeting` click-to-launch (Esc/right-click aborts). AI (diff.aiNukes: hard+specops): builds silo after 8 min, buys busters at 25.5k / tacs at 10.5k, launches within ~25s.
- **Engineer vehicle repair (2026-07-10):** engineers reuse the medic's `'heal'` order with `isVehicle` predicate + `UNIT.engineer.repair` rate (0.55/tick, faster than medic's 0.4). Idle priority: damaged buildings first, then vehicles. Right-click routing: medics take flesh, engineers take steel — one damaged target, both professions can be assigned from one mixed selection.
- **Medic (2026-07-10):** Barracks (100⬡). Unarmed (`dmg:0` + `u.dmg > 0` fire guard), NOT `isCombat` (won't auto-attack; waves send it with a move order). Heals `isFlesh` (infantry + spitters) 0.4hp/tick via `order 'heal'` (auto-acquire when idle, follows target, right-click wounded ally to assign). AI trains up to 2 after 4 min. Draws as engineer sprite + upright red-cross badge; `medic.png` OPT slot for real art. Beware the double-anchor lesson: `} else if (u.type === 'engineer') {` exists in BOTH updateUnit and drawUnit — a careless replace-all once injected canvas code into game logic.
- **Optional art slots (2026-07-09):** `OPT`/`opt()` loader — `dino_spitter/dino_nest/gunship/artillery/egg.png` in assets/sprites/ are picked up automatically when present, procedural fallback otherwise (each file independent, unlike the all-or-nothing `bodiesReady`). Spec for Bronson/artists/AI-gen: `assets/sprites/ART-WANTED.md`. New art faces UP; legacy infantry faces right.
- **Alerts (2026-07-09):** `raiseAlert` on damage to team-1 buildings/workers → red pulsing minimap pings (`alerts[]`, 2.5s) + toast/alarm throttled to one per 12s (`lastAlert`).
- **AI base-building (2026-07-09):** every 150 ticks in `aiUpdate` — fixes supply crunches, rebuilds lost barracks/factory/airpad, builds up to 2 flak if the PLAYER fields air, expands up to 3 refineries via `aiExpansionSpot()` (richest live crystal ≥500 from its dropoffs and ≥420 from any nest — the AI deliberately avoids the nest problem). `aiPlace`/`aiSpotFree` mirror canPlaceBuilding minus team-1 rules. GOTCHA fixed here: queued-supply reduce must skip `'up:'` entries (crashed once AI research queued).
- **Pacing (2026-07-09, first pass):** HQ 2200→3000, home patches 1400→1700, neutral fields +400, wave gap floor 45→55s (cadence 82−3/wave). Needs Bronson playtests before more tuning.
- **Flak (2026-07-09):** BLD `flak` (160, hotkey Y, `airOnly:1`) — targets flyers only via `nearestEnemyUnit(...,aa,airOnly)`; same anti-creep anchor rule as turrets (`isDef`); placement ghost shows range ring for any ranged building.
- **Stats + polish (2026-07-09):** `stats` {built,lost,kills,mined} → `#ov-stats` scoreboard on the game-over overlay (kills count ANY player-owned killer incl. turrets; losses = player units only). Minimap right-click = move order for selected units. Soft blip on select. Backquote = dev full-map reveal (`devReveal`).
- **Air (2026-07-09):** Gunship (240, Airpad-built; `fly:1`) — flyers skip building wall-slide/collision and only collide with other flyers (`separation`/`moveToward` guards). AA model: `noAA:1` on tank/artillery/harvester/engineer; `canAA(e)`; `nearestEnemyUnit(...aa)` filters flyers; `acquireTarget(x,y,team,range,attacker)`. Force-ordering a noAA unit at a flyer makes it give up (attack case bail). Airpad = BLD `airpad` (175, +2 supply, hotkey X); gunships benefit from Factory vehicle upgrades (IS_INF-based). AI gets an airpad per map (`M.eAir`) and trains gunships after 6 min. Gunship/airpad draw procedurally (rotor anim + ground shadow offset).
- **UI dock (2026-07-09, playtest feedback):** `#dock` = `#qpanel` (queue text + progress + rush buttons, own `lastQSig`) stacked ABOVE `#card` — production starting/finishing never shifts card buttons. Dock-level click delegation handles both panels.
- **Pause & quit (2026-07-10):** `userPaused` (P key / ⏸ chip, `#pause-banner`) gates the sim like the help modal; `quitToMenu()` (⏹ chip, two clicks within 3s) → resetWorld + start menu. Fresh startGame always clears pause.
- **Controls modal + pause (2026-07-09):** `#help` is a centered modal; `paused` gates the sim loop exactly like `!started` (render keeps running). Opens automatically in `startGame` (close = "▶ To battle"), reopens via the ? chip, Esc closes. NOTE `CC.step()` bypasses pause (deliberate, for tests).
- **Spitter cap (2026-07-09):** `SPITTER_CAP` 5 per team, enforced in `hatchSpitter`; card shows `pack n/5`; eggs are never consumed on a refused hatch.
- **Veterancy (2026-07-09):** `u.kills` → `rankOf()` (2/4/8 = Veteran/Elite/Legend, `RANK_AT`/`RANK_NAMES`); +10% dmg / −8% taken per rank folded into `weaponMult`/`armorMult`; rank 3 regens 0.05hp/tick in `updateUnit`. Kill credit flows through `bullet.src` → `damage(e,d,src)`; NOTE this also made idle units retaliate against ranged attackers (bullets used to pass `null` — deliberate improvement, guard-order dinos still don't chase). Gold chevrons via `drawRank`; promotion toast/fx for team 1.
- **Eggs (2026-07-09):** nest death drops `NEST_EGGS` (3) egg pickups (`eggs[]`, `makeEgg`). Harvester-only: right-click egg → `order 'collect'` → carry one (`u.eggCarry`) → `'returnEgg'` to the HQ ONLY (lab flavor) → `teams[t].eggs++`; auto-repeats via `u.lastEggSite` (400 radius at the clutch, else 900 near self). `hatchSpitter(hq)` (HQ card slot R, `cardActions` kind:'hatch') consumes 1 egg + 1 supply, instant spawn, player-controlled team-tinted spitter (`drawDino` uses `COLORS[u.team]`). Topbar 🥚 chip hidden until eggs exist. AI doesn't collect eggs (player-only mechanic, deliberate).
- **Maps (2026-07-09):** `MAPS` table — all-explicit authored layouts (basin = the original, gauntlet = mid-column brawl, valley = diagonal + double-nest mega center). `setup(mapKey)` + `placeBase()` are fully data-driven; add a map by adding a MAPS entry only.
- **Difficulty (2026-07-09):** `DIFFS` table (easy/normal/hard) → `diff.firstWave/waveEvery/capRate/trickle/aiUpgrades`, applied in `aiUpdate`/`waveUpdate`/`startGame`. Normal = pre-difficulty baseline exactly.
- **Start menu (2026-07-09):** `#menu` overlay in index.html; map + difficulty pickers persist to localStorage (`cc.map`/`cc.diff`). `startGame()` calls `resetWorld()` first so double-starts can't stack worlds; `started` gates the sim loop. `CC.startGame(map, diff)` + `CC.step(n)` (runs n ticks synchronously — THE way to test, since rAF sleeps in background tabs).
- **Buildings:** HQ (trains harvester/engineer, supply 20) · Barracks (infantry: marine/sniper, buildable 150) · Factory (vehicles: raider/tank, buildable 200) · Supply Depot (+8 supply, 100) · Refinery (175, harvester drop-off = expansion mechanic; must be placed near a live crystal patch, spawns a free harvester on completion) · Turret (140). Placement hotkeys T/B/V/C/G (`BUILD_MENU`); all but refinery must be within `PLACE_NEAR_BASE` of a friendly building. Supply hard cap 100 (`SUPPLY_HARD_CAP`). Enemy base includes a factory + 2 supply depots; AI runs two production queues (rax=infantry, factory=vehicles).
- **Fog of war:** tile-grid, `explored` (persistent) + `visible` (recomputed every 8 ticks). Toggle `F` / top-bar button: "remembered" (explored stays dimmed) vs "re-fogs" (blacks out when unwatched). Player targeting/clicking respects fog; AI cheats (knows player HQ location).
- **Upgrades (StarCraft-style, 2026-07-09):** `UPG` table + `teams[t].up` levels. Researched at the producing building via the same queue as units (entries `'up:<key>'`): Barracks = infWeapons/infArmor, Factory = vehWeapons/vehArmor, HQ = harvest (carry+speed). Effects via `weaponMult`/`armorMult`/`carryCap`/`effSpeed` — +12% dmg, −10% taken, +3 carry & +10% speed per level, 3 levels each. Card slots Q/W/E/R = `cardActions(b)` (units first, then research). AI researches with spare cash after 5 min. **Rush fees** (`rushProduction`): while a queue is active the card offers 2× speed (fee = half item cost, sets `b.boost=2` for the current item) or finish-now (fee = full item cost); instant respects the supply cap.
- **Game-over feel:** HQ death = triple explosion + shake 20 (normal cap 14, see `addShake`) + double boom; overlay delayed 1.4s and fx/camera keep running behind it (`frame()` else-branch).
- **Enemy AI:** mines its own economy + passive trickle, trains a mixed army capped by a ramp (`3 + 2·minutes` supply — deliberate, keeps wave 1 beatable), keeps 1 engineer after 2 min, waves at 100s then every ~70s, shrinking.
- **Controls:** drag-select, right-click smart orders (move/attack/harvest/repair), `A` attack-move, `S` stop, `H` hunker (marines), control groups `⌃1–5`, arrows/edge/minimap camera (WASD deliberately NOT camera — A/S/W/Q/E/R are command keys), `M` mute.

## Conventions & knobs

- All balance lives in the `UNIT` / `BLD` tables at the top of `game.js` plus `waveAt`/`armyCap` in the AI section. Tweak numbers there; no magic constants buried in logic (or if you add one, hoist it).
- 60 ticks/sec fixed-step loop; times in code are `seconds * 60`.
- `window.CC` is a debug handle (units, buildings, teams, fog, trainUnit, damage, commandMove, waveAt…) — used for automated browser testing; keep it working.
- Team 1 = player (teal), team 2 = enemy (red). `isCombat(u)` excludes harvester + engineer — check its call sites before adding non-combat units.

## Testing / verification

- `node --check game.js` for syntax.
- To drive it in a browser from a Claude session: serve the folder (`python3 -m http.server`) and use the preview tools + `window.CC` for assertions. **Gotcha:** a tool-spawned server may be blocked by macOS TCC from reading `~/Desktop` — if you get 404s on files that exist, copy the folder to the session scratchpad and serve from there.
- No test framework, intentionally — it's a ~3-file toy. Verify by playing/driving the real game.
